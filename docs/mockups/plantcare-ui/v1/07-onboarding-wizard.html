Wizard decisions

Same container as auth

The wizard card reuses the login/register container exactly — same radius, shadow, padding, gradient background. The user crosses from register → wizard without any visual disruption.

Progress as bars, not dots

5 horizontal bars at the top of the card fill in as you progress. The active bar gets a green glow halo; done bars are solid leaf; pending bars are mint. More scannable than numbered dots.

Skip on step 3 only

"Add your first plant" is the only optional step. Skip jumps straight to step 5 (no environment questions). If skipped, the Today screen lands with a gentle empty state pointing at the FAB.

Rooms: preset + custom

Six common rooms pre-populate (Living / Kitchen / Bedroom / Bathroom / Office / Hallway). User checks the ones they have and can add custom rooms via a dashed "+" row. Must pick at least one to continue.

Environment as segmented

Three segmented control groups stacked in one step — Light / Temperature / Humidity. Three options each, pre-default to middle. Faster than paging through three sub-questions. Note about "update anytime" reduces commitment anxiety.

Fraunces in the done speech

Monty's first voice line uses Fraunces italic 16px for impact. It's the payoff moment of the whole signup flow — the user meets their plant. Worth the typographic gravity.
