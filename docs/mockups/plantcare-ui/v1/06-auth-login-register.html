Design decisions in v11

Single screen

Both login and register are one-screen forms. No steps, no progress bar. Returning users and new users both get frictionless access. The multi-step wizard runs after registration, not inside it.

Fraunces for the heading

The auth title uses Fraunces italic — the same editorial serif we use on desktop page titles. "Welcome back" and "Join the jungle" feel like a warm invitation, not a form label.

Ambient leaf backdrop

Two large stylized monstera leaves are positioned in the background corners at 25% opacity. They anchor the brand visually without competing with the form.

Subtle focus state

The focused field gets a leaf-green border + 4px soft glow, and its icon colour shifts from ink-soft to emerald. Mint fill becomes white. Communicates interaction without chrome.

Password strength

Register has a 4-segment strength bar below the password field. Fills in with leaf green as complexity rises. Emerald when "strong". No text validation unless password is weak.

No social login (MVP)

Matches the existing technical spec — custom JWT, no OAuth providers in MVP. Phase 2 could add Google/Apple sign-in if it reduces register friction. "OR" divider is reserved for future use.
