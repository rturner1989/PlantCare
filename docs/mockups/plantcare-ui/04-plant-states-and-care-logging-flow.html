Five states of Monty

Tamagotchi decay

The same plant, drawn five ways. State is computed from how far past the watering interval Monty is — 0% just watered, 100% overdue, 150% critical. The SVG illustration swaps. You see the app getting worse as you ignore it, not just a timestamp creeping up.

How care actually gets logged

Nothing is magic. Every state change starts with a deliberate user action that creates a CareLog row. The app recomputes the plant's state from the logs. MVP uses the first two entry points. Later phases add more ways to log, all writing to the same table with a source column so we know where each log came from.
