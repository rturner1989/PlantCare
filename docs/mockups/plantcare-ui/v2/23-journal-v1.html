Timeline tab (default). Reverse-chronological feed of every event across every plant. Sticky day-group headers ("Today · Thu 24 April" in Fraunces italic) chunk the stream. Entry types carry type-specific badge colour: Care = mint/emerald, System/Weather = sunshine-tinted, Photo = coral-tinted, Milestone = sunshine-deep with a subtle horizontal gradient wash across the row. Photo entries render a 68px thumbnail inline; clicking it opens the photo in the Photos tab filtered to that plant. Toolbar above entries holds: ⊞ Filters button (mint-tinted + count pill when filters active), then the active filters as dismissible badges (plant badges carry the plant avatar, date/type badges are text only, each has an × hit-target), then a "Clear all" link. Right-side: text toggle for the stats rail ("› Hide stats" / "‹ Show stats · 48"). Rail itself collapses grid-template-columns from 1fr 260px to 1fr 0; preference persists to localStorage. On mobile the rail is absent — same toolbar pattern, badges horizontal-scroll. FAB always-on for "New entry".

Schedule tab. Full-month grid. 7 columns (Mon-Sun), 5-6 rows depending on month. Each cell carries: day number, weather icon (top-right when actionable — rain/frost/heat), stacked care dots (water/feed/photo/milestone/overdue), optional inline summary ("2 watered", "Monty rescue"). Today cell highlighted with mint fill + emerald ring. Days with entries get paper-white fill + shadow. Weather-alert days (rain forecast) get sky-blue fill + prominent icon. Overdue dots pulse coral. Legend strip at the top-right of the toolbar decodes colours. Month nav via ‹ › buttons; "Today" button jumps back. Week/Month toggle in the top-right (Week view is Phase 2). On mobile the calendar shrinks to 44px cells (date + weather + dots, no summary text) and a day-detail card below shows the selected day's full entry list. Mobile-only: tapping a day updates the detail card. Desktop-only: hover shows a popover with the same detail.

Journal locked

Journal is PlantCare's cross-cutting spine — the single source of truth for "what happened with any plant, ever". Per-plant Plant Detail · Journal tabs are filtered views of the same feed. Today's organiser drawer peeks into it. Plant Detail's hero quote is sourced from it.

  • File-tabs. Timeline (default) · Photos · Milestones · Schedule. Manila-folder vocabulary reused from Plant Detail's nested tabs. Fraunces italic, paper-on-paper, gentle shadow on active tab.
  • Entry types. Care (mint/emerald badge) · Photo (coral badge + thumbnail) · System/Weather (sunshine badge) · Milestone (sunshine-deep with row gradient wash). Colour-coded so scanning is fast.
  • Sticky day headers. "Today · Thu 24 April" in Fraunces italic, paper-deep bg, emerald strong-label. Entries chunk into day buckets so the stream reads like a diary, not a log.
  • Filter toolbar. Single ⊞ Filters button opens a grouped popover (Plant · Space · Event type · Date range · Apply/Reset footer). Active filters render alongside the button as dismissible badges with × hit-targets — "Last 30 days", plant badges carry the avatar, etc. "Clear all" link when filters applied. Mobile: popover becomes a bottom sheet.
  • Right rail (desktop only) — collapsible. Stats block (total + type breakdown), "Most active" plants (top 4), streak counter. Toggle is the text-button "Hide stats / Show stats · 48" on the right side of the filter toolbar — collapses grid-template-columns to 0. Entry count in the restore label doubles as the "info hidden here" signal. Preference persists. Rail absent on mobile — stats surface in Me region instead.
  • Schedule tab = month calendar. 7-day grid, care dots per cell, weather overlay icons on actionable days, today-ring, overdue-pulse. Week/Month toggle (Week view deferred to Phase 2). Day-detail card below on mobile; hover-popover on desktop.
  • Weather overlay. Rain/frost/heat icons ride the top-right of calendar cells. Rain days with outdoor schedule impact get sky-blue cell fill + prominent icon — same weather-pill vocabulary as House space cards.
  • New entry FAB. Universal FAB in bottom-right → opens bottom sheet (mobile) / centered dialog (desktop) for new journal entry. Photo + note + optional plant/space attribution. Same CRUD dialog vocabulary as Add Plant / Log Care.
  • Rotates LLM in Phase 2.5. Today's organiser already primes users on "journal as story". In Phase 2.5 the plant-voice bot reads the journal to respond in-character ("remember when you moved me to the south window?"). Schema must stay compatible — don't strip meta fields for Phase 1 economy.
  • Mobile dock. Journal gets its own primary dock slot (✎). Canonical template from mockup 18 — don't re-derive.
