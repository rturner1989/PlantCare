Toast locked — glass vessel + bold chip

Replaces the current TailwindUI-stock toast in the client. API unchanged (useToast / ToastContext + toast.success/error/warn/info/undo/loading). Visual treatment matches the notifications drawer vessel (glass, inset white highlight, backdrop-blur) + bold gradient-filled chip for quick category recognition.

  • Vessel. 72% paper bg · backdrop-blur 24px + saturate 1.4 · 18px border-radius · inset white highlight top + 1px white inset border. Same recipe as notifications drawer, scaled down. Quiet container so the chip can shout.
  • Bold chip. 36px circle, category gradient fill (success emerald→leaf, error coral→coral-deep, warn sunshine→sunshine-deep, info sky→sky-deep, undo mint→leaf). Radial top-left highlight wash like miniature medallions. Paper-coloured glyph inside. Drop shadow lifts it off the vessel.
  • Title em picks up category colour. "Monty watered" — "Monty" renders in emerald Fraunces italic, rest stays ink. Second accent so category reads from body text too.
  • Action button. Paper-deep bg + category-coloured text (except Undo which fills mint because time-sensitive).
  • Six variants. Success · Undo · Error · Warn · Info · Loading. Loading swaps chip for a paper-deep spinner ring.
  • Auto-dismiss windows. Success 4s · Warn 7s · Info 5s · Undo 6s · Error manual · Loading on completion.
  • Stacking. Max 3 visible, bottom-right on desktop, bottom-inset above dock on mobile. FIFO dismiss except Error/Loading.
  • A11y. role="status" for success/info/loading, role="alert" for error/warn. Screen readers announce title + meta on appearance. Action/close buttons have aria-labels matching visual text.
