Edit vs Add. Same shell (desktop dialog / mobile bottom sheet), same primitives (segmented environment rows, Fraunces-italic nickname input, paper-deep textarea). What's different: no step progress — all fields fit one panel; species row is locked; footer gets a quiet "Delete plant" text link on desktop and a bottom-of-body row on mobile. Cadence hint is the one piece of live feedback that rewards the user for tweaking environment — it surfaces inline, next to the input that caused it, only when the change materially shifts the schedule.

Edit Plant locked

Every plant's "tweak" panel. Opened from Plant Detail's edit affordance, from the House card menu, or (future) from a swipe-left gesture on the Today list. Reuses the Add Plant shell almost entirely — the only real divergence is a flat form (no wizard) plus a deliberately-quiet delete affordance.

  • Canonical primitives. Dialog shell + bottom-sheet shell copied from mockup 27. Segmented primitive uses the frozen recipe — paper-deep track, radius-md, 3px gutter, radius-xs options, active state swaps to paper + forest text + shadow-warm-sm. text-input, textarea, select, sub-label all lifted verbatim.
  • Cadence hint is the reward signal. Appears inline between Light and Temperature rows when (and only when) the environment change would shift the calculated watering cadence. Wording mirrors the Plant Detail schedule panel — "every N days" in Fraunces italic — so users recognise it as the same fact they see on the hero card.
  • NOT in this dialog. No personality chip (personality is inferred from species, never labelled). No voice-sample or plant-quote element (that's Plant Detail hero territory, mockup 22). No species rename inline — "Change species" on the locked row spawns a nested species picker reusing the Add Plant step 1 grid; destructive cadence recalc confirmation happens there, not here.
  • Delete is quiet, not aggressive. Desktop: text-link bottom-left, coral-deep, no background. Mobile: bottom-of-sheet-body row with a subtle top border and an italic "can't undo" mini-caption. Either triggers a destructive confirm variant — see open question below.
  • Open question — delete confirm pattern. Two candidates: (a) nested dialog stacks on top of Edit Plant (same shell, coral primary, "Delete Monty forever" CTA — mirrors mockup 27's "Reuse for other CRUD" note), or (b) Edit Plant closes and routes to a dedicated confirmation screen with the plant's hero image + irreversibility warnings. Option (a) keeps the user in flow; option (b) forces a beat of friction. Leaning (a) for parity with the shared dialog primitive, but worth validating with real destructive-action research before locking in.
