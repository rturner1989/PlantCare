How each step adapts per intent

One wizard, four paths. Copy, emphasis, and the Welcome-Home CTA differ by the answer given in Step 1. Same steps in the same order; per-intent config drives content. User.onboarding_intent persists after onboarding — it shapes in-app defaults later too (widget priority, prompt language, suggested jobs).

What's deliberately not in this mockup. No templated plant-voice copy on any step — Step 7 keeps the one-off welcome quote from v1 only if it survives its own review later; Step 5 Stakes is entirely visual + user-facing. No mobile adaptation — that's a separate file. No hover-only interactions beyond minor lift on buttons; every affordance works on tap so the design ports to React Native later. Illustration is a placeholder pot + leaves; real plant portraits come later (raster CDN assets per v2 §3.1i).
