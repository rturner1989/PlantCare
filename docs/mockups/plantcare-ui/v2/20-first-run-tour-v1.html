Tour variants per intent

Tour length and focus shifts based on User.onboarding_intent captured in Step 1 of onboarding. One wizard, four paths — right through the tour.

Intent Stops Sequence
🌵 Forgetful 5 greeting → rituals → swipe hint → FAB → bell (shown above)
🌱 Just starting 7 greeting → jobs rail → rituals → swipe hint → collection strip → calendar → bell
🤒 Sick plant 2 Auto-redirected to Doctor flow · 2-stop tour inside Doctor: symptom picker → tailored summary
📚 Browsing 3 greeting → Encyclopedia link → greenhouse strip

First-run arc in one picture

  • Continuity locked. Intent answered in onboarding step 1 → tour length + focus shift accordingly. User feels remembered.
  • Tour uses driver.js. Already in dependencies. Spotlight + tooltip + progress dots. Dismissable any time.
  • First care-log = loop closed. Celebration card fires the first time they swipe or wheel-water. Streak ticks visibly. App explicitly hands off ("the app's yours now").
  • Tour stored per-user. User.onboarding_tour_completed_at. Replayable from Me region.
  • Five stops on Forgetful. Short enough to land. Long enough to cover the core loop (rituals + swipe + FAB + organiser). Other intents adjusted.
  • Spotlight ring pulses. 2.4s ease-in-out loop on outline colour — the contemplative tempo for tour overlays. Paired vocabulary: the radial wheel's urgent spoke in mockup 22 pulses faster, at 2.2s. Two distinct motion vocabularies — contemplative tour vs urgent action — kept separate on purpose.
  • Tooltip auto-flip. Classes arrow-top / arrow-bottom / arrow-left / arrow-right set the default arrow side. Production uses driver.js's edge detection to flip the tooltip to the opposite side when it would overflow the viewport — the arrow class follows.
  • Keyboard / gesture parity. Arrow keys advance tour on desktop. Esc dismisses. Mobile = tap anywhere outside to advance, tap tooltip's Skip to exit.
  • Reduced-motion fallback. No pulse on prefers-reduced-motion; tooltip transitions become opacity cross-fade instead of slide.
