The zero state matters most

Empty states are the first thing a user sees on a region they haven't populated yet. They need to sell the region's value, not apologise for emptiness. Warm copy, Fraunces italic heading, one primary action, optional secondary. Same shell on every region — medallion + headline + subtext + action row. Matches the v2 "collection-first, companion-shaped" positioning.

Pattern. Every empty state uses the same shell — centered medallion (140px circle with gradient + soft shadow + highlight wash), Fraunces italic headline with gradient em emphasis, subtext (max 380px, 14px ink-soft), one primary action, optional secondary or ghost action. Illustrations are emoji-forward (no custom SVG illustrations — Fraunces + warm gradients carry the personality). Region label in the top-left (ink-ink pill) makes the pattern-library context obvious; in the app itself this label is replaced by the region's normal head.

Empty states locked

Empty states are a shared primitive, not 8 bespoke designs. One React component EmptyState (already exists in the client per TICKET-009 pickup) with props: icon, headline, body, primaryAction, secondaryAction, ghostAction, variant (for illustration gradient: mint/coral/sunshine/sky/forest). Each region calls it with its own copy.

  • Copy leans into value, not apology. "Your greenhouse is waiting" not "You have no plants yet". "Everyone's thriving. No rituals today." not "No tasks". Fraunces italic headline carries warmth; subtext is factual 14px body.
  • Primary action points forward. "＋ Add your first plant" appears on Today · House · Journal · Me first-run empties — all point to the same FAB flow (mockup 27). Consistency means users don't have to relearn where to add plants.
  • Distinct copy for distinct zeros. "No plants yet" (first-run) vs "All caught up" (positive empty) vs "No results for query" (search empty) vs "No entries match filters" (filtered empty) — each gets its own headline + actions. Don't collapse into one generic "nothing here" card.
  • Positive empty ≠ apology. "All caught up" shows a forest-gradient medallion (the only dark-variant), streak chip, soft secondary actions. It's a reward state, not a void. Users should feel good arriving here.
  • Filtered empty has its own shape. When filters zero out a region (Journal, Encyclopedia), primary action is always "Clear filters" or "Widen" — not "Add a plant". The problem is the filter, not the collection.
  • Search empty links to browsing. Encyclopedia "couldn't find X" primary is "Browse popular" (fallback engagement), secondary is "Request this species" (captures the unmet need, potentially seeds Perenual-fetch backfill).
  • Illustration gradient by emotional tone. mint = neutral / encouraging. sunshine = warm / milestone / pending. coral = narrative / storytelling. sky = informational / search / reference. forest = reward / accomplished (dark-variant, use sparingly).
  • Two empties for Today. First-run ("greenhouse is waiting") and caught-up ("everyone's thriving") are two different empties — don't confuse them. The app knows which: plants.count == 0 → first-run; plants.count > 0 && tasks.empty? → caught-up.
  • No personality badge, no voice sample. Copy tone carries the warmth. No 🎭 pill, no first-person plant quote. Personality stays inferred, per the v2 rule.
  • Reduced motion friendly. No animation on the medallion or illustration — these stay static. Focus stays on copy + action.
  • A11y: Illustration emoji marked aria-hidden, headline is semantic h2 (within region context), primary action has explicit aria-label duplicating button text (for icons-first buttons).
