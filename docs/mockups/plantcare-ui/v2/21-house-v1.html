Search is a filter, not an overlay. Tap ⌕ → input grows from right (220ms spring). User types → current region's data narrows live. Non-matching rooms / plants / entries / articles hide; matching rows highlight the matched segment. No context-switching, no separate results screen. Cancel (✕) restores full list.

Same behaviour on desktop. Sidebar search (or ⌘F) filters the current region in place. ⌘K is separate (keyboard-only) and opens a command palette for global navigation + quick actions — a different beast, invoked on demand, not the default search.

Per-region scope is implicit. On House = filter plants/rooms. On Journal = filter entries. Switch region, filter resets.

House region locked

  • Two views. Rooms (default, spaces-as-cards) and List (flat sortable). User toggles via pill at top-right. Preference persists to User.house_view.
  • Habitat is Phase 3. Third toggle slot visible but disabled with small "P3" badge. Sets expectations for the 3D view coming later.
  • Rooms grid attention. Rooms with overdue plants show a coral "N overdue" pill + coral ring on the specific mini-avatar. Glanceable.
  • Indoor vs outdoor. Outdoor spaces use sunshine-tinted icons (not mint/emerald). Visual cue without needing labels.
  • List grouping. Plants grouped by space with Fraunces italic group headers. Sticky on scroll. Same vocabulary as the grid view — consistency of space iconography.
  • Weather-skip hint. Outdoor plants show "Skipping — rain" as next-care when Phase-2 weather integration kicks in. Renders as coral-ish italic meta.
  • Search is shared. Toolbar search works across both views — typing filters rooms (by name) or plants (by name/species) depending on active view.
  • FAB = Add Plant always. Same universal action. Bottom-right on desktop, centre dock slot on mobile.
  • Long-press = radial wheel. On plant mini-avatars (grid), plant rows (list), or any plant anywhere. Same as Today.
  • Add a space. Appears as a dashed-plus card at the end of the rooms grid on desktop. Opens Add Space dialog (bottom sheet mobile / modal desktop). Mobile doesn't show it inline — lives in the "..." menu top-right or accessed via an empty-state prompt when user has few spaces.
