Floats top-right with 14px inset over the main area. No scrim — Mac-notification-centre style. Header: Fraunces italic "Your notifications" + subtitle "3 unread · 12 total this week" + "Mark all as read" dotted-underline link. Filter chip row below: All · Unread · Care · Rescues · Milestones · Weather · System (matches Journal entry-type vocabulary). List grouped by day with sticky Fraunces italic day headers (also backdrop-blurred). Each item: 36px avatar with category badge (coloured 16px circle overlayed bottom-right) + meta row (plant name in emerald + event-type badge in category colour) + body text + optional action hint (💧 Water now → for actionable items) + timestamp. On mobile: right-slide full-width drawer (88% of screen width) — matches the three-motion vocabulary (right-slide = info panel). Dock dimmed to 0.6 opacity to indicate modal context.

Notifications locked

Notifications are a sibling surface to Journal — same event types (care/rescue/milestone/weather/system), same colour coding, same day grouping. Difference: Journal is the permanent archive, Notifications is the transient inbox. Read items stay 7 days then vanish from the drawer but persist in Journal forever.

  • Bell in two places. Desktop: sidebar top-right (next to logo). Mobile: topbar top-right. Active state = mint fill + concentric ring + coral unread-count badge. Same primitive (NotificationBell), different parent.
  • Floating glass organiser (desktop). 380px wide, translucent paper-bg + backdrop-blur, floats top-right with 14px inset over the main area. Same vocabulary as Today's organiser (mockup 17). No scrim — Mac-notification-centre style. Click-outside or Escape closes. Content behind stays readable + interactive-looking (soft "this is paused" impression without full dim).
  • Right-slide drawer (mobile). Full-width panel slides in from the right — same as mockup 18's organiser. Three-motion vocabulary (right-slide = info panel, bottom-sheet = CRUD, radial = per-plant). Same content as desktop, different container.
  • Category coding matches Journal. Care = sky-blue · Rescue = coral · Milestone = sunshine-deep · Weather = sky-blue · System = emerald. Exact same colours as Journal entry-type badges. Users learn once.
  • Actionable items get an action hint. "💧 Water now →" / "📷 Add a photo →" — emerald text, 11px, under the body. Tapping the item navigates to the relevant surface (Plant Detail with action wheel cued, etc). Non-actionable items (rescued, streak unlocked) skip the hint.
  • Unread state. Coral dot on the left (6px circle with soft halo) + mint tint on the row bg. Count in the bell badge. Items auto-mark-read on tap; "Mark all as read" link in the header.
  • Filter chips. All · Unread · Care · Rescues · Milestones · Weather · System. Counts on active chip. Horizontal-scroll on mobile. Same primitive as Journal's filter popover chips, reused inline here (filter surface is smaller than Journal's, so no popover needed).
  • Day grouping. Today · Yesterday · Earlier this week · This month (older automatically collapsed). Fraunces italic sticky headers match Journal. After 7 days, items roll out of the Notifications drawer; Journal is the permanent record.
  • Footer has settings link. "⚙ Notification settings →" routes to Me · Notifications. Users who want to tune cadence don't have to hunt.
  • Push notifications reuse this schema. Phase 2 ships Web Push — each push maps to an item in this drawer (same icon · category badge · body copy · action hint). When the user taps the push, it opens the app and animates the matching item in the drawer.
  • A11y. Drawer is a <dialog> with focus trap; Escape + scrim-click close. Bell button announces count ("3 unread notifications"). List items are real buttons with aria-label summarising the item. Unread state announced via aria-describedby.
