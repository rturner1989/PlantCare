Error states locked

Three shells covering every reasonable failure mode. Full-page (404, 500) for routing / server errors. Offline banner for network loss. Inline region error for single-widget failures. All share the circle-motif + Fraunces italic vocabulary — medallion + h1 + Fraunces italic em + action row.

  • Full-page shell. Centered column: 140px medallion with Fraunces italic glyph (404 number, ! for 500, custom glyph for others) → h1 with gradient em → body paragraph → action row (primary CTA + secondary + ghost). Gradient paper medallion for neutral errors (404), coral-gradient for server errors (500).
  • Copy voice. Warm, first-person-plural when blame is on us ("our server tripped over a root"). No error codes in the main body. No stack traces ever. Primary CTA uses a destination ("Back to Today") not just "Go home".
  • Offline banner. Top-of-viewport sticky, coral-tinted glass vessel. Fires on navigator.onLine = false or API network-error. Non-blocking — cached content stays interactive. Rows below get a "cached · will sync when online" meta line. Optimistic actions queue and replay on reconnect. Auto-dismiss on recovery (+ success toast "Back online — synced N actions").
  • Inline region error. For single-widget failures. Paper card + medallion chip (sunshine for recoverable, coral for harder) + Fraunces italic title. Inline "Retry" / "Dismiss". Use when the rest of the page is fine — don't escalate to full-page error for a broken widget.
  • Toast for transient errors. Short-lived failures that the user can't fix from the error surface (save-failed, rate-limit) go to the toast.error variant (mockup 31). Don't use inline-error for something that'll fix itself on retry.
  • Upgrade-required. Phase-2.5 LLM features gated behind payment use the inline-error shell with a sunshine chip + "Upgrade to unlock" primary. Doesn't belong here long-term — a dedicated billing / upgrade mockup would own it.
  • Maintenance / scheduled-downtime. Full-page shell with a paper medallion + calendar glyph, copy "We're planting new beds" + expected back-at time. Rare enough to not need its own mockup yet.
  • A11y. Full-page errors are role="main" with a live-region role="status" announcing the headline. Offline banner is role="alert" so screen readers interrupt. Inline-error sections use aria-errormessage / aria-describedby to link to the affected widget where possible.
  • Error boundary wrapping. Every React region (Today / House / Journal / etc) is wrapped in a top-level ErrorBoundary that renders the inline-error shell if the region throws. Full-page 500 only for app-root errors that escape every boundary.
