Log Care locked

Long-form care logger opened from a plant's overflow menu. Three care events (water / feed / photo / note / prune / move / other) route through the same dialog — fields reveal conditionally. Everything about it rhymes with the Add Plant shell so the CRUD vocabulary stays one primitive deep.

  • Conditional Amount field. Only renders for Water or Feed. For Photo / Note / Prune / Move / Other, the field is removed from the DOM entirely — no greyed-out placeholder, no "N/A" state. The sub-label meta (only shown for Water · Feed) documents the rule where it lives, so future readers aren't surprised when the field vanishes.
  • CTA label is dynamic. Log watering / Log feeding / Log photo / Log note / Log pruning / Log move / Log care — the verb follows the care type. Mobile keeps the same label; no truncation needed at 7-char max ("watering" fits). Reinforces that the primary action is specific, not generic.
  • Distinct from the radial wheel and CareConfirmDialog. The House radial wheel's one-tap water is for when you're walking past a plant and just want to record that you did the thing — no modal, no form. The shipped CareConfirmDialog (Today ritual row) confirms a pre-selected care task in a single tap. This dialog is for the deliberate, context-rich log — "I did X, took a photo, noticed Y, and it was 3 days ago not today". Three surfaces for three levels of intent — don't collapse them.
  • Care-type segmented is 7 options. Desktop fits all 7 in a single row (56px-wide cells with stacked icon + label). Mobile uses a 4-col grid that wraps to 2 rows (4 + 3), using the canonical .segmented primitive — icons sit above labels, consistent with the display: inline-flex; flex-direction: column pattern. No horizontal scroll — everything is visible at a glance.
  • Plant identity stays visible. Header carries a small avatar chip (🌿 Monty) so the user never has to wonder which plant they're logging against. On mobile the eyebrow reads Log care · Monty for the same reason — plant name inline, not buried.
  • Photo dropzone → attached swap. Empty state is a dashed dropzone with Choose/Camera buttons. Attached state is a coral-tinted thumbnail placeholder with filename, metadata, and Replace/Remove actions. The coral tint is a mockup convention — real implementation shows the actual image. Keeping photo attachment inline (not a separate step) preserves the single-screen form.
  • Notes typography = body font, not Fraunces. This is an operational note, not a voice-sample quote. Fraunces italic is reserved for the Plant Detail hero quote (mockup 22) — keeping that distinction means voice-samples feel earned and typographically special, not generic.
  • Open question · backdated entries. The "Choose another date" path opens a native date picker on mobile and a small calendar popover on desktop (not drawn here). Unresolved: how far back can you backdate a log? Unlimited feels right, but it breaks streaks and schedule recalculation if used carelessly. Follow-up on the backend care_logs spec.
  • Open question · "Other" care type. "Other" is a catch-all for unusual events (pest treatment, humidity tray, staking). Does it need a free-text subtype field, or does the Note suffice? Current design: Note is enough. Will revisit if users start writing "Actually this was a pest treatment" in Notes repeatedly.
  • Success flow. On submit, the dialog closes with a scale-down + toast: "Logged watering for Monty · 250ml" (scene A) or "Photo added to Monty's journal" (scene B). The plant's Plant Detail journal timeline refreshes. If the log satisfies a Today ritual, that ritual row checks off simultaneously.
