Sidebar responsive rules

Three states cover every desktop viewport. Which state is active is entirely driven by window width — no user toggle between states. Users can toggle the drawer open/closed below 768px via the burger, but they cannot choose rail-at-1400px or full-at-700px. Auto-matching the breakpoint keeps the implementation honest.

Window width Sidebar state Width used Labels How to toggle
≥ 1200px Full 240px fixed Always shown N/A — always expanded
768 – 1199px Icon rail 64px fixed Tooltip on hover N/A — auto-collapsed, not toggleable
< 768px Drawer 0px (hidden) → 260px (open overlay) Always shown when open Burger button · Escape · scrim tap
  • Auto-state, not manual. No "pin/unpin" toggle. Users get the right state for their viewport without fiddling. This keeps markup decisions clean — no persisted "sidebar-mode" preference to sync across devices.
  • Vocabulary shared across states. Nav items render the same data — icon, label, count, active state — just with different chrome around them. Implementation-wise this is one React component Sidebar with a mode prop ("full" | "rail" | "drawer") picked by a useMediaQuery / useViewportSize hook.
  • Icon rail keeps count badges. Overdue or due-today counts are critical scan signals; dropping them would make the rail decorative. Badges show as a top-right coral chip on the icon tile (or emerald when the tile is active).
  • Drawer reuses Full layout. The drawer panel IS the Full sidebar; it's just positioned as a 260px overlay instead of a grid column. One component, two positions — no duplicate markup.
  • Tooltip on hover in Icon rail. Black ink-coloured pill (labelled "House" etc.) appears to the right of the icon on hover / focus. Keyboard-accessible; reads naturally on screen readers because the underlying button keeps an aria-label that matches the tooltip text.
  • Command-search keyboard shortcut works in all three states. ⌘K / Ctrl-K opens the search dialog regardless of whether the search field is expanded, collapsed to an icon, or hidden behind the drawer. Shortcut is the stable API.
  • Mobile phones don't see any of these. Phone viewports (≤480px) use the dock from mockup 18 instead of a sidebar entirely — different interaction model. The drawer state is for narrow laptop / portrait tablet, not phones.
  • Current app is mid-way. The client already has the drawer state (TICKET-019 md-breakpoint burger). v2 adds the icon-rail state in between, and tightens the drawer's scrim + label consistency.
