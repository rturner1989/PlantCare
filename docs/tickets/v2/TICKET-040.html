Summary

Plant Detail (/plants/:id) is currently a placeholder. This ticket builds the v2 page per mockup 22: hero with always-open RadialWheel + medallion centre, four CareRings (Water · Feed · Light · Mood), three panels (Schedule · Recent care · Environment), top-level Segment tabs (Care · Species · Journal), nested FileTabs inside the Journal tab. In one sentence: the portrait-and-timeline page where every plant lives — rings own state, panels own config and history, no facts duplicated between them.

Requirements

  1. Visiting /plants/:id renders the full v2 Plant Detail page (mockup 22), not a placeholder.
  2. Hero shows the RadialWheel always-open with the medallion centre showing the plant's avatar — same plant-portrait recipe used elsewhere (PlantTile / QuickDialog / DayRituals wheel). Urgent ring fires when plant is overdue.
  3. Four CareRings render side-by-side: Water (sky), Feed (mint), Light (sunshine), Mood (coral when struggling, mint when thriving). Each shows label + value only — no subtitles, no rotation.
  4. Schedule panel reads "Every N days — {reasoning}" sourced from species + space env modifiers. NO countdown / overdue text — that's the rings' job.
  5. Recent care panel shows the last N care logs in chronological order.
  6. Environment panel exposes editable Light / Temperature / Humidity Segments — saves to space (server's SpacesController#update already handles env levels).
  7. Top-level Segment tabs (Care / Species / Journal) switch the main panel content. Care is the default; Species shows species-level info; Journal embeds nested FileTabs.
  8. Nested Journal FileTabs (Timeline · Photos · Milestones · Schedule) filter the journal feed to this plant via useJournal({ plant_id }). (Journal endpoint may still be stubbed pre-R5 — wire what's available, render placeholder for missing data.)
  9. Overflow menu surfaces Edit · Log care · Doctor · Delete actions. R12 dialogs aren't built yet — wire the menu items to placeholders or no-ops, leave a comment.
  10. Desktop scroll model: page scrolls as one; hero scrolls away; Segment tab-bar sticks to top once it reaches the viewport edge (iOS large-title pattern).

React concepts worth looking up

Compound components for FileTabs. The FileTabs primitive uses the same compound shape as Card / Menu — FileTabs.Panel is a static property on the parent. Consumer authoring API stays declarative: <FileTabs activeId="timeline" onChange={setTab}><FileTabs.Panel id="timeline">…</FileTabs.Panel>…</FileTabs>. Rails analog: ViewComponent slots — single-paired (renders_one) for the active panel, plus multi (renders_many) for the tab strip itself.
Sticky positioning + scroll containment. position: sticky works only when the parent has a scrollable region AND the sticky element has a top offset. Plant Detail's main column scrolls; the Segment tab-bar gets top-0 sticky z-10. Make sure no ancestor has overflow: hidden in the wrong place — sticky breaks silently if the nearest scroll container is wrong.
SVG circular progress arc math. CareRing uses a stroke-dashoffset trick — a circle with circumference C, set stroke-dasharray={C} and stroke-dashoffset={C * (1 - value/max)}. Animate dashoffset to fill. strokeLinecap="round" for the soft cap. Same idea as the ProgressRing primitive already in the repo — copy the math, change the semantics (state vs progress).
Lazy-loaded route. Plant Detail is heavy (RadialWheel + CareRings + tabs + nested tabs + journal feed). Wire it via React.lazy(() => import('./pages/PlantDetail')) + the existing AppLayout Suspense fallback — same pattern other v2 pages use. Keeps the initial bundle lean.
WAI-ARIA tablist pattern. Segment + FileTabs both implement the same tabs/tabpanel/tab roles + arrow-key keyboard nav. Single source of truth: each tab has role="tab" + aria-selected; the panel has role="tabpanel" + aria-labelledby. Roving tabindex moves focus across tabs. The existing SegmentedControl already has this; FileTabs mirrors it.

Notes & gotchas

  • Rings own state, panels own config + history. Don't print "Next water in 3 days" in both the Water ring AND the Schedule panel. Schedule says the rule ("Every 7 days — bright light bump"); the ring shows the live countdown.
  • Don't build R12 dialogs. Overflow menu items wire to placeholder no-ops with a TODO comment. R12 ticket lands them.
  • Mood ring needs a derived metric. If the backend doesn't expose a "mood" value, derive client-side from water_status + feed_status (overdue → struggling, due_today/soon → watchful, healthy → thriving). Same logic PlantTile already uses for its mood dot — extract if both consumers want it, otherwise inline.
  • RadialWheel size. Plant Detail wants 420px desktop hero. The wheel currently caps at lg (440px) which is close. Use size="lg" or extend SIZES if the wheel needs a fresh "hero" preset.
  • Journal feed may not be wired yet. R5 Journal region (TICKET-041) builds useJournal. If it's not present, scaffold a stub hook in this ticket that returns an empty array — drop it when 041 lands. Comment the stub explicitly.
  • Closing checklist: ./scripts/lint.sh && ./scripts/run_tests.sh, run /vercel-react-best-practices + /accessibility, /comment-audit, then open a PR with feat(v2): prefix.
