Requirements

  1. Sidebar renders three distinct visual states based on viewport width: Full at ≥1200px, Icon-rail between 768–1199px, Drawer below 768px.
  2. Sidebar shows five nav items in every state: Today, House, Journal, Encyclopedia, Me — in that order. Each links to its route and reflects the active route via aria-current="page".
  3. Icon-rail state shows count badges (coral chip top-right of icon) for items that have counts; rail items reveal a dark tooltip with the label on hover or keyboard focus.
  4. Drawer state opens via a burger trigger in the top bar, dims the page with a scrim, and closes on Escape, scrim tap, or close button.
  5. Mobile dock (<md) shows four nav items + centre Add FAB: Today · House · [+ FAB] · Journal · Me. Encyclopedia is not in the dock.
  6. MobileTopBar (52px sticky) renders at <md and holds: burger, logo, spacer, bell button, avatar chip linking to Me.
  7. Bell button has an accessible name ("Notifications") even when no badge count is rendered.
  8. PlantAvatar no longer renders the 🎭 personality emoji or any personality text label.
  9. Routing: /discover is removed; /encyclopedia, /journal, /notifications are added. Each new route renders the existing PlaceholderPage component until its region ticket lands.
  10. Sidebar mode pick survives a window resize without remount glitch — switching from 1400 → 1000 → 600 changes states cleanly.
  11. Reduced-motion respected — no resize-driven animations for users with the OS preference set.

React concepts worth looking up

Reading window dimensions in React. window.innerWidth only exists in the browser, not during SSR/initial render. The pattern: initialise state with a sensible default (e.g. 'full'), then update inside a useEffect that subscribes to the resize event. Debounce so you're not re-rendering the entire layout on every pixel of a drag. Always return a cleanup function from the effect that removes the listener — otherwise every navigation leaks one.
StrictMode double-fire of effects. In dev, React intentionally mounts → unmounts → re-mounts every component once to catch missing cleanup. If your resize listener is added in useEffect without a cleanup, you'll see two listeners attached after a refresh. The cleanup function (returning () => window.removeEventListener(...)) is what makes the effect survive StrictMode without leaking.
NavLink + active state. React Router's NavLink already exposes an isActive flag in its className/style render-prop. Use it to apply the active styling AND to set aria-current="page" on the underlying anchor — accessibility wants both the visual cue and the semantic attribute. end prop matters: to="/" matches every path unless you set end, otherwise Today highlights as active on every page.
Accessible name for icon-only buttons. A button with just an SVG inside has no accessible name — screen readers will read "button" with no context. aria-label="Notifications" is the fix on the bell. For the icon-rail nav items, a tooltip is a visual hint; the button still needs an aria-label (or visually-hidden label) so the SR experience matches the sighted one. Tooltips are a UX nicety, not an a11y substitute.
When NOT to memo. Vercel's rerender-simple-expression-in-memo rule: don't wrap trivial computed values in useMemo. The viewport-size hook returns a tiny string; the components that consume it aren't doing expensive work. Memoising the mode pick is cargo-cult. Ship the simple version; the React Compiler (when it's on) handles the cases that actually matter.

Notes & gotchas

  • Don't wire the bell to a real endpoint here. B4 Notifications (TICKET-023) is the right home for that. Stub the button + leave a single-line TODO referencing TICKET-023, no useNotifications hook stub.
  • Don't build a cmd-K dialog. The search pill is a visual placeholder. Keyboard shortcut not wired. A future ticket owns the dialog + indexer.
  • The drawer state already exists from TICKET-019's md-breakpoint burger work. Reuse the existing AnimatePresence + scrim pattern in Sidebar.jsx; don't reinvent it. The new work is the icon-rail branch between drawer and full.
  • Watch the lg vs md breakpoint in the existing Sidebar/Dock — current code switches at lg (1024px) and md (768px). Mockup 26 specifies the rail boundary at 1200px. Use a custom value (useViewportSize's thresholds), not a Tailwind breakpoint, for the full↔rail switch — Tailwind's xl default is 1280px which is too high.
  • NavLink active-class typo trap. NavLink takes a function-as-className. If you pass a string instead it never highlights. Read the existing Sidebar.jsx pattern for the right shape before retyping.
  • The PlantAvatar 🎭 strip will break tests that assert on the emoji. Update those tests in the same commit, don't leave them red.
  • Don't rename the /me route. It's already wired and the auth flow redirects to it. Keep the path; just leave the placeholder page in place.
  • When you're done: run ./scripts/lint.sh && ./scripts/run_tests.sh client, run the three review skills (/vercel-react-best-practices, /accessibility, /dhh-rails-reviewer on any backend touch — none expected here), then /pre-commit + /comment-audit, commit on this branch (ticket-037-sidebar-dock-nav), and open the PR with a chore(v2): or feat(v2): prefix.
