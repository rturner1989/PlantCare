What you're building

Five pieces — three ui/ primitives, one root-level domain component, and the rewrite of pages/Today.jsx that consumes them.

Componentcomponents/ui/WeatherPill.jsx

A pill-shaped status surface with a circular icon medallion + label + optional detail line. Comes in three layout variants (strip for Today's full-width band, calendar for 7-day cells, group for House space-group headers) and three colour schemes (sky default, frost, heat). Pulses via motion/react when urgent — reduced-motion respected. Lands first because it's the smallest primitive and the calendar inside Organiser depends on it.

Refactorcomponents/ui/GroupCard.jsx

Pulled inline-out-of NotificationsDrawer.jsx into a shared primitive. Owns the Card.Header / Card.Body / Card.Footer chrome, the optional viewAll={{ count, onClick }} expand affordance, and the sequenced expand-animation state machine (view-all swoosh → sibling exit → card layout move → items reveal → header swap). NotificationsDrawer becomes a consumer of this primitive in the same PR. Today's Organiser uses it for "Jobs queue" + "Weather forecast" + "This week's calendar" + "Streak".

Componentcomponents/ui/RadialWheel.jsx

Six action spokes (Water · Feed · Photo · Note · Doctor · Move) fanning around a centre medallion that holds the plant's avatar. Long-press on a Today urgent ritual row opens the wheel; pointer-up closes it; tapping a spoke fires the action. Open animation staggers spokes outward via motion/react; the primary spoke gets a 2.2s ease-in-out coral-glow pulse when urgent. Sized 300px on mobile, 420px on desktop (R4 Plant Detail will use it always-open at the larger size). Keyboard nav: Tab into centre, Arrow keys cycle spokes, Enter fires, Esc closes.

Componentcomponents/Organiser.jsx

Today's "context companion" — a column on desktop, a bottom-sheet drawer on mobile (the same Drawer primitive R8 already ships). Holds state-snapshot widgets: weather forecast (WeatherPill strip variant), this week's care calendar (7 cells with weather + care dots), current streak, jobs queue. Triggered by the ✎ pencil button on Today's mobile top bar. Does not duplicate Notifications — no journal preview card, no event list, no read state. Per project_organiser_vs_notifications_scope.md: Organiser is "what's true right now", Notifications is "what just happened".

Rewritepages/Today.jsx

Ground-up rebuild. Order top-to-bottom: collection strip (every plant as a small mood-decorated avatar, coral ring on those needing care — not a hero, just a presence), jobs rail (named-verb cards from the dashboard payload), weekly calendar (7-day strip with weather + care dots), ritual rows (56px SummarySlab height — swipe-right on mobile to mark complete, long-press on urgent rows to open RadialWheel). Desktop adds the Organiser as a right-side column; mobile reaches it via the ✎ button. The current HeroCard is demoted to "rescue mission" surface that conditionally renders only when there's a genuinely urgent overdue plant; it stops being the page hero.

Requirements

  1. Today is collection-first. A user opening Today sees every plant they own at a glance via small mood-decorated avatars, not a single hero card. Plants needing care wear a coral ring; the rest are calm. The HeroCard surface only appears when there's a genuinely overdue rescue scenario.
  2. Rituals are completable inline. Each ritual row is a 56px slab with the plant's avatar, the action verb, and a relative-time pill. On mobile, swiping the row right marks the action complete (logs care + animates the row out). On desktop, an explicit complete button does the same. Keyboard users get Space/Enter as the swipe equivalent.
  3. Urgent ritual rows expose the RadialWheel. Long-press (or right-click on desktop) on an urgent row opens the wheel anchored to the row's avatar. Tapping a spoke fires the chosen action; the wheel closes on pointer-up, Esc, or selecting a spoke. Reduced-motion users get a non-animated reveal.
  4. Weather is consistent across surfaces. The same WeatherPill primitive renders the Today weather strip, the calendar cells inside Organiser, and (in future tickets) House outdoor cards + Journal calendar. No four-variant drift like v1.
  5. Organiser is state, not events. The Organiser column holds present-tense widgets: weather, this week's calendar, current streak, jobs queue. It does not show recent activity (that's Notifications) or historical entries (that's Journal). The split is enforced by the content, not just the trigger.
  6. Notifications drawer continues to work. R8's NotificationsDrawer keeps its visual + functional behaviour after the GroupCard extraction — same animation, same expand-back flow, same a11y semantics. The PR includes a regression test that proves it.
  7. Both Today and the primitives respect reduced-motion. Swipe-row exit animation, RadialWheel open stagger + urgent pulse, WeatherPill urgent pulse, Organiser open transition — all gated on useReducedMotion().
  8. The collection grid stays responsive even with a big plant list. A user with 60+ plants doesn't see the search/filter UI lag while typing. Use useDeferredValue on any expensive derived list (mood-grouping, filter results) so input stays snappy.
  9. Tests cover the new surfaces. WeatherPill, RadialWheel, GroupCard, Organiser each get a Vitest test file. Today gets a Playwright spec covering: page load, complete-a-ritual flow, open-the-wheel flow, open-the-organiser flow.

React concepts worth looking up

useDeferredValue for big lists. When the collection strip filters or re-groups based on user input (or a derived computation runs over 60+ plants), you can wrap the input/dep in useDeferredValue so React keeps the typing responsive and runs the expensive render in the background. It's the React 18+ idiom — newer than your hobby learning. Rails analogy: like marking a query as "low priority" so the controller doesn't block on it before responding to the page render.
Framer Motion drag gestures + popLayout. Swipe-to-complete is a motion/react drag handler with a threshold check (offset.x > N px or velocity > Y). On exit, the row animates with AnimatePresence mode="popLayout" so the rows below close the gap smoothly. The same library powers the RadialWheel stagger reveal — you've used Framer in R8 and R9, this is more of the same.
Long-press vs click on touch. Pointer events fire before touch gestures resolve — building a "long-press opens, tap completes" interaction means tracking pointer-down time + cancelling the long-press timer if pointer-up comes first. Don't conflate it with React's synthetic onClick; pointer events give you the timing primitives you need.
useReducedMotion is a hook, not a CSS rule. motion/react exposes useReducedMotion() which reads the OS preference; pass { duration: 0 } as the transition when it's true. Already used across R8/R9 — same pattern. The CSS @media (prefers-reduced-motion) approach works for pure-CSS animation, but Framer-driven motion needs the hook because the values come from JS.
When NOT to memoise. Vercel's rerender-simple-expression-in-memo rule says: if the computation is a primitive lookup or trivial expression, useMemo costs more than it saves. The collection strip's per-avatar style derivation (mood → ring colour) is exactly that. Do not wrap it. The React Compiler is doing the work for you when it makes sense.
The "compound child" slot pattern. WeatherPill has a single, paired-with-parent slot shape (icon + label + detail) — that's a props contract (renders_one in ViewComponent terms). GroupCard takes a body of variable items — that's children (content). Don't mix patterns within a primitive. CLAUDE.md "Slot patterns" section is canonical; default to props for single+paired, children for variable+freeform.

Notes & gotchas

  • Don't introduce a new Drawer primitive. R8 already shipped components/ui/Drawer.jsx (composes Dialog with placement="right"). The Organiser's mobile sheet rides on it. Mode toggle (column vs drawer) lives on Organiser itself, not on a new primitive.
  • NotificationsDrawer must keep working. The GroupCard extraction is a refactor — visual + animation behaviour identical. The R8 Playwright drawer spec is your regression net. Run it after the extraction commit and before any other change touches GroupCard.
  • The .weather-strip / .sum-row.weather / .weather-alert / .sg-weather drift from v1 mockups is the explicit known-debt cleanup F4 was created for. Fold all four shapes into WeatherPill variants from day one — don't ship the primitive without the group variant just because R3 (House) hasn't started yet.
  • Weather + Streak are placeholder data. Build a small adapter (hooks/useWeather.js returning hardcoded { today, week }) so the swap to real data later is one file, not a sweep across components. Don't block this PR on backend work.
  • RadialWheel positioning is anchored to the row, not the viewport. Use a portal-rendered wheel positioned via the row's getBoundingClientRect (or a popper-style anchor lib if it gets messy). Don't try to make it a child of the row — overflow + z-index will fight you.
  • Long-press + swipe-to-complete share the same pointer surface. Resolve the gesture priority: swipe wins if horizontal movement exceeds N px before the long-press timer fires; long-press wins if pointer stays still past 500ms. Test on mobile viewport with a touch emulator.
  • StrictMode double-fire on the long-press timer effect. If you use useEffect to set up pointer listeners, the cleanup must clear the timer + remove listeners cleanly. R8's cable subscription effect pattern is a good reference.
  • Per pair-programming style memo (feedback_pair_programming_style.md) — work each step through together. Don't batch the whole ticket to an agent. Rob drives learning + spots pivot/cleanup. Test-only steps stage + advance without pause; production-code steps get reviewed.
  • Per feedback_rework_review_skills.md — bake /vercel-react-best-practices + /accessibility into AC. /dhh-rails-reviewer is N/A — this is frontend-only. Reviews run before opening the PR, blocking findings fixed in the same branch.
  • When you're done: run ./scripts/lint.sh && ./scripts/run_tests.sh client, commit on a feature branch (ticket-038-today-v2), then open a PR. PR title gets the feat(v2): prefix per feedback_pr_title_v2_prefix.md; no Claude attribution per feedback_pr_body_no_claude_footer.md.
