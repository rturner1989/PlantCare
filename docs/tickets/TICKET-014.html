TICKET-014

Sidebar live counts + profile route

Open Low

Summary

The desktop Sidebar renders a Today badge count of 2 and a House badge count of 12 — both hardcoded. Logout currently lives in the sidebar footer next to the user avatar; it should live on the Me profile page where the rest of the account-level actions belong. Two long-standing TODOs in Sidebar.jsx point at both gaps.

In one sentence: wire the badges to real data and move logout off the nav chrome.

Where this fits

TICKET-008 built the sidebar/dock primitives with placeholder numbers and a placeholder logout spot; the comments have carried TODO(ticket 14) tags ever since. TICKET-009 and TICKET-010 (Today + House screens) bring the real data hooks that this ticket consumes. When the Me screen lands as part of the broader tasks 11–14 polish, the avatar/logout section moves with it.

Sequencing-wise, this only makes sense after TICKET-009 (dashboard) and TICKET-010 (house) are in — those are where overdue and plants_count live. Doing it earlier means inventing placeholder contracts we'd throw away.

What you're building

extendclient/src/components/Sidebar.jsx
Replace the hardcoded count: 2 and count: 12 with live values from useDashboard() (overdue count for Today) and useRooms() or usePlants() for total plants (House). Counts hide when the value is zero so the badge doesn't draw attention with nothing to say.
hookclient/src/hooks/useDashboard.js
Already exists. Surface a derived overdueCount in the hook's return shape so every consumer (sidebar, Today) doesn't re-count. Cheap: data?.overdue?.length ?? 0.
pageclient/src/pages/Me.jsx
New profile page at /me. Hosts name/email display, "Log out" button (moves in from Sidebar), and a placeholder for future account settings. Route already exists as a PlaceholderPage — swap it for this.
extendclient/src/components/Sidebar.jsx (footer)
The footer stays: avatar + name + email + link to /me. The logout button goes away. Clicking the avatar goes to the profile page and logout happens there. Dock already behaves this way via the Me tab — sidebar catches up.

Requirements

  1. Sidebar's Today badge shows the actual number of overdue tasks, not 2.
  2. Sidebar's House badge shows the actual total plant count, not 12.
  3. Badges render as zero-hidden — a user with no overdue tasks sees no number, not a pill saying "0".
  4. Logout is no longer a sidebar element. A new /me page renders the logout button alongside the user's profile info.
  5. Clicking the avatar in the sidebar footer navigates to /me.
  6. Both TODO(ticket 14) comments in Sidebar.jsx are deleted (they self-resolve).

Acceptance criteria

Fresh signup with no plants: Today and House badges render no number. Sidebar footer avatar is present; no separate logout icon.
Add one overdue task: Today badge becomes 1. Add a plant: House badge becomes 1. Numbers update live after mutations (invalidation via TanStack Query already wired).
Click avatar: lands on /me. Click "Log out": session ends and redirects to /login with a toast (same behaviour as today's sidebar button).
./scripts/lint.sh and ./scripts/run_tests.sh client both pass. Playwright spec for the new Me page + logout flow added.

Notes & gotchas

  • Dock already has a Me tab — don't duplicate it. The mobile logout flow routes via Dock → Me page, same as desktop after this lands.
  • Don't over-build the Me page. Name, email, logout. Anything else (password change, email change, delete account) is out of scope — those earn their own tickets when the product needs them.
  • Zero-hidden badges: check how Badge handles falsy counts. Easiest is to render null when count is 0 from the mapping code, rather than teaching Badge new tricks.
